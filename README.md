# 🛒 Online Marketplace Database

База данных онлайн-маркетплейса (типа Ozon) на PostgreSQL.  
**17 таблиц** в 3НФ, **50 SQL-запросов**, **Web UI** для просмотра и выполнения запросов, Docker для запуска.

---

## 🚀 Быстрый старт

### Требования
- [Docker](https://docs.docker.com/get-docker/) и Docker Compose

### Запуск

```bash
# Запустить всё (PostgreSQL + Web UI)
make up
```

После запуска:
- 🌐 **Web UI**: [http://localhost:8080](http://localhost:8080)
- 🐘 **psql консоль**: `make psql`

### Все команды

```bash
make help       # Справка по всем командам
make up         # Запустить PostgreSQL + Web UI
make psql       # Открыть интерактивную консоль psql
make queries    # Выполнить все 50 запросов
make query SQL="SELECT * FROM products"   # Выполнить один запрос
make init       # Пересоздать схему и перезагрузить данные
make down       # Остановить контейнеры
make clean      # Остановить и удалить все данные
make logs       # Логи PostgreSQL
make web-logs   # Логи Web UI
```

## 🌐 Web UI

Web-интерфейс на Flask с четырьмя страницами:

### 📊 Dashboard (`/`)
- Список всех 17 таблиц с количеством записей
- Быстрый доступ к примерам запросов по категориям
- Клик по таблице → просмотр данных

### 🗄️ Schema (`/schema`)
- **Граф связей** — визуальная диаграмма всех таблиц и FK-связей
- Структура каждой таблицы (столбцы, типы, PK, NOT NULL)
- Список всех Foreign Keys

### ⚡ SQL Query (`/query`)
- Редактор SQL-запросов
- Боковая панель с 30+ примерами запросов по категориям
- Ctrl+Enter для выполнения
- Результаты в виде таблицы

### 📋 Table Viewer (`/table/<name>`)
- Просмотр структуры и данных любой таблицы

## ✅ Нормализация (3НФ)

### 1НФ — Первая нормальная форма
- Все атрибуты атомарны (нет массивов, вложенных структур)
- Каждая таблица имеет первичный ключ (`SERIAL PRIMARY KEY`)
- Нет повторяющихся групп

### 2НФ — Вторая нормальная форма
- Нет частичных зависимостей от составного ключа
- Пример: `order_items(order_id, product_id)` → `quantity`, `unit_price` зависят от всего ключа

### 3НФ — Третья нормальная форма
- Нет транзитивных зависимостей
- `categories` и `subcategories` — отдельные справочники (не поле в `products`)
- `cities` отделены от `customers`, `warehouses`, `pickup_points`
- `warehouses` отделены от `products`
- `couriers` отделены от `orders`

## 🐳 Как это работает

1. **Первый запуск** (`make up`): Docker создаёт volume `pgdata`, PostgreSQL выполняет скрипты из `init/` — создаёт 17 таблиц и заполняет тестовыми данными. Это происходит **один раз**.

2. **Последующие запуски**: PostgreSQL видит, что volume уже содержит данные, и **пропускает** инициализацию. Контейнер стартует с готовой базой за ~3 секунды.

3. **Сброс данных**: `make clean` удаляет volume, после чего следующий `make up` пересоздаст всё заново.
